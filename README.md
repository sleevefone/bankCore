# bank-core

`bank-core` 是给 `pay-hello` 对接的内部核心系统第一版架构项目。

当前阶段先不急着写实现，先把架构边界固定下来，避免后续把“支付编排”和“核心账务”混在一起。

核心目标：

- 作为支付系统的内部账务核心，对外提供稳定的记账能力
- 承接支付成功后的入账、出账、冻结、解冻、冲正等动作
- 保证幂等、可追踪、可补偿、可审计
- 覆盖核心账务审计维度，包括账号、账号序号、客户号、借贷方向、余额、科目、利息详情与计提
- 所有表统一采用“自增逻辑主键 `id` + 独立业务主键/业务唯一键”的建模规范
- 为后续对账、结算、运营修复、风控联动留出扩展点

文档入口：

- [核心系统架构设计](./docs/architecture.zh-CN.md)

本地开发约束：

- 本项目统一使用 `JDK 21` 构建与运行
- Maven 命令必须运行在 `Java 21` 下，否则会被 `maven-enforcer-plugin` 直接拦截
- 如果本机同时装了多个 JDK，先执行 `export JAVA_HOME=$(/usr/libexec/java_home -v 21)` 再运行 `mvn clean package`
- IDEA 需要同时把 `Project SDK` 和 `Maven Runner JRE` 都切到 `JDK 21`

建议的系统边界：

- `pay-hello` 负责支付订单编排、渠道交互、回调处理、支付状态汇总
- `bank-core` 负责账户、账号序号、客户号、余额、科目、账务流水、分录、利息计提、幂等、记账结果查询与审计

第一版推荐只做最小闭环：

- 接收支付系统的核心记账请求
- 支持 `PAY_IN`
- 支持按 `requestId` 或 `bizOrderId` 查询结果
- 保证重复请求不重复记账
- 为后续利息与计提扩展预留账户、科目和审计数据结构
